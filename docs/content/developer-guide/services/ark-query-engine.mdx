# Ark Query Engine

Built-in execution engine that runs the LLM turn loop, tool execution, team orchestration, memory, and streaming for Ark queries.

- Deployed as a sidecar in the controller pod
- Communicates with the controller via A2A protocol
- Handles built-in agent, team, model, and tool execution (agents with an `executionEngine` are dispatched directly by the controller)
- Streams chunks directly to ark-broker

## Architecture

The controller delegates query execution to the query engine via A2A `SendMessage`. The engine reads CRDs using its own K8s client, executes the turn loop through the existing `genai` package, and returns results.

```
Controller Pod
┌──────────────────────────────────────────────────────┐
│  Controller          A2A         Query Engine         │
│  (reconciler) ──────────────►  (sidecar :9090)       │
│       │        ◄──────────────       │               │
│  - watch CRs                   - agent loop          │
│  - resolve target              - team orchestration   │
│  - write status                - tool execution       │
│                                - LLM providers        │
│                                - memory load/save     │
│                                - stream → ark-broker  │
└──────────────────────────────────────────────────────┘
```

## Configuration

The query engine is enabled by default in the Helm chart.

### Helm Values

```yaml
queryEngine:
  enabled: true
  container:
    image:
      repository: ghcr.io/mckinsey/agents-at-scale-ark/ark-query-engine
      pullPolicy: IfNotPresent
    port: 9090
    resources:
      limits:
        cpu: 500m
        memory: 256Mi
      requests:
        cpu: 10m
        memory: 64Mi
```

### Controller Flag

The controller connects to the engine via the `--query-engine-addr` flag (default: `http://localhost:9090`). In sidecar mode this is localhost. In dev mode it points to the engine's K8s Service.

## Local Development

In dev mode, the query engine runs as a separate pod with independent code sync and restart.

```bash
cd ark
devspace dev
```

DevSpace deploys the engine using the `ark-query-engine-dev` chart at `charts/ark-query-engine-dev/`. The controller's `--query-engine-addr` is set to `http://ark-query-engine.ark-system:9090`.

Changes to engine code restart only the engine pod. Changes to controller code restart only the controller pod.

## A2A Message Contract

The controller sends the same fat metadata contract used by external execution engines (Python SDK's `BaseExecutor`), extended with a query reference:

```json
{
  "role": "user",
  "parts": [{ "text": "<user input>" }],
  "metadata": {
    "ark.mckinsey.com/execution-engine": {
      "agent": { "name": "...", "namespace": "..." },
      "tools": [],
      "history": [],
      "query": { "name": "q-123", "namespace": "default" }
    }
  }
}
```

The engine reads the Query CR and target CRDs using its K8s client for tool execution, MCP discovery, memory, and streaming config.
